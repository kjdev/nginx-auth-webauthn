# Registration page example (`examples/register/`)

> ⚠️ **For development and learning only**. For production use the bundled CLI
> [`tools/admin/auth-webauthn-admin`](../../tools/admin/). This Python server
> favors simplicity and does not meet operational requirements such as
> authentication, rate limiting, CSRF protection or duplicate-user control.

A reference implementation where a credential generated by the browser's
`navigator.credentials.create()` is attestation-verified by a Python server
(Flask + python-fido2) and written to Redis in exactly the format the NGINX
module reads.

```
register/
├── index.html        registration UI
├── register.js       /register/begin → navigator.credentials.create() → /register/complete
├── register.py       Flask + python-fido2 + redis-py (verification, Redis write)
└── requirements.txt
```

## Differences from the CLI

| | `register.py` (this example) | `auth-webauthn-admin register` (CLI) |
|---|---|---|
| challenge issuance | server issues and keeps in session | none (clientData is a hash only) |
| origin/type/challenge verification | performed by python-fido2 | not supported (hash only) |
| use case | learning, browser verification | operations, scripted registration |

Both share the same Redis schema (`docs/ARCHITECTURE.md` §5.1). Credentials
registered with `register.py` can be inspected with `auth-webauthn-admin list`
/ `show` and used as-is for login from `examples/login/`.

## Setup

```sh
cd examples/register
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt

# have Redis (Valkey) running
export REDIS_URL=redis://127.0.0.1:6379/0
export WEBAUTHN_RP_ID=localhost
export WEBAUTHN_ORIGIN=http://localhost:5000

python register.py        # listens on http://127.0.0.1:5000/
```

Open `http://localhost:5000/` in a browser, enter a user ID and click "Register
a passkey". `localhost` is a WebAuthn secure-context exception, so plain HTTP
works.

Confirm the registration with the CLI:

```sh
tools/admin/auth-webauthn-admin list --user-id alice
```

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `WEBAUTHN_RP_ID` | `localhost` | RP ID. Match the NGINX `auth_webauthn_rp_id`. |
| `WEBAUTHN_RP_NAME` | `nginx-auth-webauthn example` | RP display name |
| `WEBAUTHN_ORIGIN` | `http://localhost:5000` | Allowed origin. The origin of the URL that calls `navigator.credentials.create()`. |
| `WEBAUTHN_KEY_PREFIX` | `webauthn:` | Redis key prefix. Match the NGINX `auth_webauthn_redis_key_prefix`. |
| `REDIS_URL` | `redis://127.0.0.1:6379/0` | Redis connection |
| `FLASK_SECRET_KEY` | (random per process) | Flask session signing key |

## Implementation notes

- **Requires a discoverable credential (resident key)** (`residentKey:
  required`). Because `examples/login/` sends an empty `allowCredentials`, a
  discoverable credential the authenticator can resolve to a user handle is
  required.
- **COSE → DER conversion**: the public key is reconstructed with
  `cryptography` from python-fido2's `CoseKey` (a dict of COSE labels) and
  stored as `SubjectPublicKeyInfo` DER. This is byte-identical to the DER the
  CLI produces with OpenSSL `i2d_PUBKEY`. EC2 (ES256/384/512) / RSA (RS256) /
  OKP (Ed25519) are supported.
- Attestation is delegated to python-fido2's default verifiers
  (`attestation=direct`).
